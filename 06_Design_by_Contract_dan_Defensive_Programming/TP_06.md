```
const assert = require('assert');

function divide(a, b) {
  assert(typeof a === 'number' && typeof b === 'number', 'Nilai harus bilangan bulat');

  assert(b !== 0, 'Tidak bisa pembagian dengan nol');

  return a / b;
}
```
function divide(a, b) {
  if (typeof a !== "number" || typeof b !== "number") {
    throw new TypeError("Nilai harus bilangan bulat");
  }

  if (b === 0) {
    throw new Error("Tidak bisa pembagian dengan nol");
  }

  return a / b;
}

try {
  const result = divide(10, 2);
  console.log("Hasilnya adalah:", result);
} catch (error) {
  console.error("Error:", error);
}
```
Menurutmu, kapankah kita saatnya menggunakan asersi atau eksepsi untuk fungsi seperti ini di atas? Apakah kita harus sepenuhnya asersi, atau sepenuhnya eksepsi? Lakukan riset dan berikan jawabannya dalam bentuk esai minimal 300 kata.

## Jawaban:

Untuk menentukan kapan menggunakan asersi dan eksepsi pada fungsi seperti pembagian, kita bisa mulai dari memahami tujuan dari masing-masing pendekatan terlebih dahulu. Tapi di potongang koding ini fungsi pembagian terlihat sederhana, tapi tetap butuh validasi supaya tidak terjadi error saat dijalankan.

Asersi itu biasanya digunakan saat proses development, jadi lebih ke arah ngecek apakah asumsi yang dibuat programmer itu benar atau tidak. Misalnya kita sudah yakin kalau input yang masuk pasti angka dan tidak mungkin nol, maka asersi bisa dipakai buat memastikan hal itu. Kalau ternyata asersi gagal, berarti ada kesalahan dalam logika program yang kita buat. Jadi bisa dibilang asersi ini lebih cocok buat debugging, bukan untuk menangani kesalahan dari user.

Sedangkan eksepsi itu digunakan untuk menangani error yang memang mungkin terjadi saat program dipakai. Dalam kondisi nyata, kita tidak bisa menjamin kalau user selalu memasukkan input yang benar. Bisa saja user memasukkan string, null, atau bahkan angka nol untuk pembagi. Nah, di sinilah eksepsi berperan, yaitu untuk menangani error tersebut supaya program tidak langsung berhenti dan masih bisa memberikan pesan yang jelas.

Kalau dilihat dari kebutuhan fungsi pembagian tadi, sebenarnya eksepsi lebih cocok digunakan karena fungsi tersebut berhubungan langsung dengan input user. Jadi kemungkinan error dari luar itu besar, dan harus ditangani dengan baik.

Tapi bukan berarti asersi nggak berguna. Asersi tetap bisa dipakai saat development untuk membantu memastikan bahwa bagian internal program berjalan sesuai harapan. Jadi misalnya sebelum fungsi dipakai secara luas, kita bisa pakai asersi untuk ngecek kondisi-kondisi tertentu.

Jadi kesimpulannya, kita tidak harus memilih sepenuhnya asersi atau sepenuhnya eksepsi. Lebih baik keduanya digunakan sesuai kebutuhan. Asersi dipakai saat development untuk debugging, sedangkan eksepsi dipakai saat program berjalan untuk menangani error dari user atau kondisi yang tidak terduga. Dengan cara ini, kode jadi lebih aman, jelas, dan tetap mudah dipahami, terutama untuk programmer yang masih belajar.
